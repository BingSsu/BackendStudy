# HTTP 프로토콜

1. HTTP 프로토콜이란
```
인터넷상에서 데이터를 주고 받기 위한 서버/클라이언트 모델을 따르는 프로토콜이다. 애플리케이션 레벨의 프로토콜로 TCP/IP위에서 작동한다. 어떤 종류의 데이터든지 전송할 수 있도록 설계되어 있으며 HTTP로 보낼 수 있는 데이터는 HTML문서, 이미지, 동영상, 오디오, 텍스트 문서 등 여러종류가 있다.
하이퍼텍스트 기반으로(Hypertext) 데이터를 전송하겠다(Transfer) = 링크기반으로 데이터에 접속하겠다는 의미이다.

프로토콜 : 컴퓨터나 원거리 통신장비 사이에서 메시지를 주고받는 양식 또는 규칙
```

2. 작동 방식
```
HTTP는 서버/클라이언트 모델을 따른다.
클라이언트에서 요청을 보내면 서버는 요청을 처리해서 응답 한다.

클라이언트 : 서버에 요청하는 클라이언트 소프트웨어(IE, Chrome, FireFox, Safari...)가 설치된 컴퓨터를 이용한다.
클라이언트 URL을 이용해서 서버에 접속하고, 데이터를 요청할 수 있다.

서버 : 클라이언트 요청을 받아서, 요청을 해석하고 응답을 하는 소프트웨어가 설치된 컴퓨터 등이 서버 소프트웨어다.
웹 서버는 보통 표준인 80번 포트로 서비스 한다.
```

3. Connectless & Stateless
```
HTTP는 Connectless 방식을 사용한다. 즉 서버에 요청을 하고 요청해서 응답을 받으면 연결을 끊어버린다

장점 : 불특정 다수를 대상으로 하는 서비스에 적합
=> 수십만명이 웹 서비스를 이용하더라도 접속 유지는 최소한으로 할 수 있기 때문에, 더 많은 유저의 요청을 받을 수 있다.

단점 : 연결을 끊어버리기에 클라이언트의 이전 상태를 알 수 없다.
이러한 HTTP의 특징을 Stateless라고 하는데, Connectless로부터 파생되는 특징이다.

클라이언트의 이전 상태를 알 수 없게되면 당장 웹 서비스를 하는데 문제가 생긴다. 클라이언트가 과거에 로그인해도 정보를 유지할 수 없다. HTTP는 이러한 문제를 Cookie를 이용해서 이 문제를 활용하고 있다.
```

4. URL
```
클라이언트 소프트웨어는 URL을 이용하여 자원위 위치를 찾는다.
URL과 HTTP의 차이가 있는데 HTTP는 전송 프로토콜이고 URL은 자원의 위치를 나타내기 위해 사용한다.

예를 들어 https://www.example.com/index.php 이라는 URL이 있다. 이 URL은 자원의 접근을 위해 https방식의 프로토콜을 사용하며 자원의 안터넷 상에서 위치는 www.example.com이다 이는 ip주소로 변환 되므로 ip 주소로 서버를 찾을 수 있다. index.php는 요청할 자원의 이름이다.

이렇게 "프로토콜", "위치", "자원명"으로 어디에 있던지 자원에 접근할 수 있다.
https://www.example.com/index.php라 함은 https방식을 사용해서 www.example.com서버에 index.php라는 자원을 요청하는 것이다.
```

5. Method
```
메서드는 요청의 종류를 서버에 알려주기 위해서 사용한다.
다음은 요청에 사용할 수 있는 메서드들이다.

GET : 정보를 요청하기 위해 사용 (SELECT)
POST : 정보를 올리기 위해 사용(INSERT)
PUT : 정보를 업데이트 하기 위해 사용(UPDATE)
DELETE : 정보를 삭제하기 위해 사용(DELETE)
HEAD : 헤더 정보만 요청. 해당 자원이 존재하는지 혹은 서버에 문제가 없는지를 확인하기 위해 사용
OPTIONS : 웹 서버가 지원하는 메서드의 종류를 요청
TRACE : 클라이언트의 요청을 그대로 반환한다. 예컨데 echo 서비스로 서버 상태를 확인하기 위한 목적으로 주로 사용
```
